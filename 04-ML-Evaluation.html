การวัดประสิทธิภาพโมเดล Machine Learning

เมตริกและฟังก์ชันการสูญเสียสำหรับ Regression และ Classification

การวัดประสิทธิภาพโมเดล (Model Evaluation Metrics)

การวัดประสิทธิภาพโมเดลเป็นขั้นตอนสำคัญในการพัฒนาโมเดล Machine Learning เพื่อประเมินว่าโมเดลสามารถทำนายได้แม่นยำเพียงใด โดยเมตริกที่ใช้จะแตกต่างกันไปตามประเภทของงาน

เมตริกสำหรับ Regression

ใช้วัดความแม่นยำในการทำนายค่าต่อเนื่อง โดยวัดความแตกต่างระหว่างค่าที่ทำนายได้กับค่าจริง

  • • MSE (Mean Squared Error)
  • • RMSE (Root Mean Squared Error)
  • • MAE (Mean Absolute Error)
  • • R-squared (Coefficient of Determination)

เมตริกสำหรับ Classification

ใช้วัดความแม่นยำในการจำแนกประเภทหรือคลาส โดยเปรียบเทียบคลาสที่ทำนายได้กับคลาสจริง

  • • Accuracy (ความแม่นยำ)
  • • Precision (ความเที่ยง)
  • • Recall (ความไว)
  • • F1-score
  • • ROC-AUC

เมตริกสำหรับ Regression

เมตริกเหล่านี้ใช้วัดความแม่นยำของโมเดลในการทำนายค่าต่อเนื่อง โดยวัดความแตกต่างระหว่างค่าที่ทำนายได้ (ŷ) กับค่าจริง (y)

MSE (Mean Squared Error)

ค่าเฉลี่ยของกำลังสองของความแตกต่างระหว่างค่าที่ทำนายได้กับค่าจริง

MSE = (1/n) * Σ(y_i - ŷ_i)²

RMSE (Root Mean Squared Error)

รากที่สองของค่าเฉลี่ยของกำลังสองของความแตกต่างระหว่างค่าที่ทำนายได้กับค่าจริง

RMSE = √[(1/n) * Σ(y_i - ŷ_i)²]

MAE (Mean Absolute Error)

ค่าเฉลี่ยของค่าสัมบูรณ์ของความแตกต่างระหว่างค่าที่ทำนายได้กับค่าจริง

MAE = (1/n) * Σ|y_i - ŷ_i|

R-squared (Coefficient of Determination)

สัดส่วนของความแปรปรวนในตัวแปรตามที่สามารถอธิบายได้ด้วยโมเดล

R² = 1 - [Σ(y_i - ŷ_i)² / Σ(y_i - ȳ)²]

MSE (Mean Squared Error) - รายละเอียด

คำอธิบาย: MSE คือค่าเฉลี่ยของกำลังสองของความแตกต่างระหว่างค่าที่ทำนายได้กับค่าจริง

สูตร:

MSE = (1/n) * Σ(y_i - ŷ_i)²

โดยที่:

  • n คือจำนวนตัวอย่างทั้งหมด
  • y_i คือค่าจริงของตัวอย่างที่ i
  • ŷ_i คือค่าที่ทำนายได้ของตัวอย่างที่ i

การแปลผล:

  • ค่า MSE ยิ่งน้อยยิ่งดี (ค่าต่ำสุดคือ 0)
  • ให้น้ำหนักกับความผิดพลาดขนาดใหญ่มากกว่าความผิดพลาดขนาดเล็ก เนื่องจากมีการยกกำลังสอง
  • หน่วยของ MSE คือหน่วยของตัวแปรตามยกกำลังสอง

ข้อดี:

  • คำนวณง่าย
  • มีคุณสมบัติทางคณิตศาสตร์ที่ดี เช่น สามารถหาอนุพันธ์ได้
  • ลงโทษความผิดพลาดขนาดใหญ่มากกว่าความผิดพลาดขนาดเล็ก

ข้อเสีย:

  • ไวต่อค่า Outlier มาก
  • หน่วยของ MSE เป็นหน่วยของตัวแปรตามยกกำลังสอง ทำให้ตีความได้ยาก

การใช้งาน:

  • ใช้เป็นฟังก์ชันการสูญเสีย (Loss Function) ในการฝึกโมเดล Regression
  • ใช้เปรียบเทียบประสิทธิภาพระหว่างโมเดลต่างๆ
  • เหมาะกับกรณีที่ต้องการลงโทษความผิดพลาดขนาดใหญ่มากกว่าความผิดพลาดขนาดเล็ก

RMSE (Root Mean Squared Error) - รายละเอียด

คำอธิบาย: RMSE คือรากที่สองของค่าเฉลี่ยของกำลังสองของความแตกต่างระหว่างค่าที่ทำนายได้กับค่าจริง

สูตร:

RMSE = √[(1/n) * Σ(y_i - ŷ_i)²]

โดยที่:

  • n คือจำนวนตัวอย่างทั้งหมด
  • y_i คือค่าจริงของตัวอย่างที่ i
  • ŷ_i คือค่าที่ทำนายได้ของตัวอย่างที่ i

การแปลผล:

  • ค่า RMSE ยิ่งน้อยยิ่งดี (ค่าต่ำสุดคือ 0)
  • ให้น้ำหนักกับความผิดพลาดขนาดใหญ่มากกว่าความผิดพลาดขนาดเล็ก
  • หน่วยของ RMSE เป็นหน่วยเดียวกับตัวแปรตาม ทำให้ตีความได้ง่ายกว่า MSE

ข้อดี:

  • มีหน่วยเดียวกับตัวแปรตาม ทำให้ตีความได้ง่าย
  • ลงโทษความผิดพลาดขนาดใหญ่มากกว่าความผิดพลาดขนาดเล็ก
  • เป็นที่นิยมใช้ในการเปรียบเทียบโมเดล

ข้อเสีย:

  • ไวต่อค่า Outlier มาก
  • การคำนวณซับซ้อนกว่า MSE เล็กน้อย

การใช้งาน:

  • ใช้เปรียบเทียบประสิทธิภาพระหว่างโมเดลต่างๆ
  • ใช้ประเมินความแม่นยำของโมเดลในหน่วยเดียวกับตัวแปรตาม
  • นิยมใช้ในงานพยากรณ์ เช่น พยากรณ์ราคา พยากรณ์ยอดขาย

MAE (Mean Absolute Error) - รายละเอียด

คำอธิบาย: MAE คือค่าเฉลี่ยของค่าสัมบูรณ์ของความแตกต่างระหว่างค่าที่ทำนายได้กับค่าจริง

สูตร:

MAE = (1/n) * Σ|y_i - ŷ_i|

โดยที่:

  • n คือจำนวนตัวอย่างทั้งหมด
  • y_i คือค่าจริงของตัวอย่างที่ i
  • ŷ_i คือค่าที่ทำนายได้ของตัวอย่างที่ i

การแปลผล:

  • ค่า MAE ยิ่งน้อยยิ่งดี (ค่าต่ำสุดคือ 0)
  • ให้น้ำหนักกับความผิดพลาดทุกขนาดเท่ากัน
  • หน่วยของ MAE เป็นหน่วยเดียวกับตัวแปรตาม

ข้อดี:

  • มีหน่วยเดียวกับตัวแปรตาม ทำให้ตีความได้ง่าย
  • ทนทานต่อค่า Outlier มากกว่า MSE และ RMSE
  • ให้น้ำหนักกับความผิดพลาดทุกขนาดเท่ากัน

ข้อเสีย:

  • ไม่ลงโทษความผิดพลาดขนาดใหญ่มากกว่าความผิดพลาดขนาดเล็ก
  • อนุพันธ์ไม่ต่อเนื่องที่จุด 0 ทำให้อาจมีปัญหาในการหาค่าต่ำสุด

การใช้งาน:

  • ใช้เป็นฟังก์ชันการสูญเสีย (Loss Function) ในการฝึกโมเดล Regression
  • ใช้เปรียบเทียบประสิทธิภาพระหว่างโมเดลต่างๆ
  • เหมาะกับกรณีที่ไม่ต้องการลงโทษความผิดพลาดขนาดใหญ่มากกว่าความผิดพลาดขนาดเล็ก

R-squared (Coefficient of Determination) - รายละเอียด

คำอธิบาย: R-squared คือสัดส่วนของความแปรปรวนในตัวแปรตามที่สามารถอธิบายได้ด้วยโมเดล

สูตร:

R² = 1 - [Σ(y_i - ŷ_i)² / Σ(y_i - ȳ)²]

โดยที่:

  • y_i คือค่าจริงของตัวอย่างที่ i
  • ŷ_i คือค่าที่ทำนายได้ของตัวอย่างที่ i
  • ȳ คือค่าเฉลี่ยของค่าจริงทั้งหมด

การแปลผล:

  • ค่า R² อยู่ระหว่าง 0 ถึง 1 (ในบางกรณีอาจติดลบได้)
  • ค่า R² ยิ่งมากยิ่งดี (ค่าสูงสุดคือ 1)
  • R² = 1 หมายถึงโมเดลสามารถอธิบายความแปรปรวนของข้อมูลได้ทั้งหมด
  • R² = 0 หมายถึงโมเดลไม่สามารถอธิบายความแปรปรวนของข้อมูลได้เลย

ข้อดี:

  • ไม่มีหน่วย ทำให้เปรียบเทียบระหว่างโมเดลต่างๆ ได้ง่าย
  • ตีความได้ง่ายในรูปของสัดส่วนความแปรปรวนที่อธิบายได้
  • เป็นที่นิยมและเข้าใจกันอย่างแพร่หลาย

ข้อเสีย:

  • ค่า R² จะเพิ่มขึ้นเสมอเมื่อเพิ่มตัวแปรอิสระ แม้ว่าตัวแปรนั้นจะไม่มีความสัมพันธ์กับตัวแปรตามก็ตาม
  • ไม่สามารถบอกได้ว่าโมเดลมีความแม่นยำในการทำนายมากน้อยเพียงใด
  • อาจให้ค่าติดลบได้ในบางกรณี

การใช้งาน:

  • ใช้วัดความสามารถในการอธิบายความแปรปรวนของข้อมูลของโมเดล
  • ใช้เปรียบเทียบประสิทธิภาพระหว่างโมเดลต่างๆ
  • ใช้ในการรายงานผลการวิเคราะห์การถดถอย

เปรียบเทียบเมตริกสำหรับ Regression

เมตริก ช่วงค่า ค่าที่ดีที่สุด ความไวต่อ Outlier หน่วย
MSE 0 ถึง ∞ 0 สูงมาก หน่วยของตัวแปรตาม²
RMSE 0 ถึง ∞ 0 สูง หน่วยของตัวแปรตาม
MAE 0 ถึง ∞ 0 ปานกลาง หน่วยของตัวแปรตาม
R-squared -∞ ถึง 1 1 ปานกลาง ไม่มีหน่วย

เมตริกสำหรับ Classification

เมตริกเหล่านี้ใช้วัดความแม่นยำของโมเดลในการจำแนกประเภทหรือคลาส โดยเปรียบเทียบคลาสที่ทำนายได้กับคลาสจริง

Confusion Matrix (เมทริกซ์ความสับสน)

Confusion Matrix เป็นตารางที่แสดงผลการจำแนกประเภทของโมเดล โดยเปรียบเทียบระหว่างคลาสที่ทำนายได้กับคลาสจริง ซึ่งเป็นพื้นฐานในการคำนวณเมตริกอื่นๆ

True Positive (TP)
False Positive (FP)
False Negative (FN)
True Negative (TN)

True Positive (TP): ทำนายว่าเป็นคลาสบวก และเป็นคลาสบวกจริง

False Positive (FP): ทำนายว่าเป็นคลาสบวก แต่เป็นคลาสลบจริง (Type I Error)

False Negative (FN): ทำนายว่าเป็นคลาสลบ แต่เป็นคลาสบวกจริง (Type II Error)

True Negative (TN): ทำนายว่าเป็นคลาสลบ และเป็นคลาสลบจริง

Accuracy (ความแม่นยำ)

สัดส่วนของการทำนายที่ถูกต้องทั้งหมด

Accuracy = (TP + TN) / (TP + TN + FP + FN)

Precision (ความเที่ยง)

สัดส่วนของการทำนายคลาสบวกที่ถูกต้อง

Precision = TP / (TP + FP)

Recall (ความไว)

สัดส่วนของคลาสบวกจริงที่ถูกทำนายว่าเป็นคลาสบวก

Recall = TP / (TP + FN)

F1-score

ค่าเฉลี่ยฮาร์โมนิกของ Precision และ Recall

F1 = 2 * (Precision * Recall) / (Precision + Recall)

ROC-AUC

พื้นที่ใต้กราฟ ROC (Receiver Operating Characteristic)

AUC = พื้นที่ใต้กราฟ ROC

Accuracy (ความแม่นยำ) - รายละเอียด

คำอธิบาย: Accuracy คือสัดส่วนของการทำนายที่ถูกต้องทั้งหมด

สูตร:

Accuracy = (TP + TN) / (TP + TN + FP + FN)

โดยที่:

  • TP (True Positive) คือจำนวนตัวอย่างที่ทำนายว่าเป็นคลาสบวก และเป็นคลาสบวกจริง
  • TN (True Negative) คือจำนวนตัวอย่างที่ทำนายว่าเป็นคลาสลบ และเป็นคลาสลบจริง
  • FP (False Positive) คือจำนวนตัวอย่างที่ทำนายว่าเป็นคลาสบวก แต่เป็นคลาสลบจริง
  • FN (False Negative) คือจำนวนตัวอย่างที่ทำนายว่าเป็นคลาสลบ แต่เป็นคลาสบวกจริง

การแปลผล:

  • ค่า Accuracy อยู่ระหว่าง 0 ถึง 1 (หรือ 0% ถึง 100%)
  • ค่า Accuracy ยิ่งมากยิ่งดี (ค่าสูงสุดคือ 1 หรือ 100%)
  • Accuracy = 1 หมายถึงโมเดลทำนายได้ถูกต้องทั้งหมด
  • Accuracy = 0 หมายถึงโมเดลทำนายผิดทั้งหมด

ข้อดี:

  • เข้าใจง่าย
  • คำนวณง่าย
  • เหมาะสำหรับปัญหาที่มีความสมดุลของคลาส

ข้อเสีย:

  • ไม่เหมาะสำหรับปัญหาที่มีความไม่สมดุลของคลาส (Class Imbalance)
  • อาจทำให้เข้าใจผิดเกี่ยวกับประสิทธิภาพของโมเดลในกรณีที่คลาสไม่สมดุล

การใช้งาน:

  • ใช้วัดประสิทธิภาพโดยรวมของโมเดล
  • เหมาะสำหรับปัญหาที่มีความสมดุลของคลาส
  • ใช้เป็นเมตริกเริ่มต้นในการประเมินโมเดล

Precision (ความเที่ยง) - รายละเอียด

คำอธิบาย: Precision คือสัดส่วนของการทำนายคลาสบวกที่ถูกต้อง

สูตร:

Precision = TP / (TP + FP)

โดยที่:

  • TP (True Positive) คือจำนวนตัวอย่างที่ทำนายว่าเป็นคลาสบวก และเป็นคลาสบวกจริง
  • FP (False Positive) คือจำนวนตัวอย่างที่ทำนายว่าเป็นคลาสบวก แต่เป็นคลาสลบจริง

การแปลผล:

  • ค่า Precision อยู่ระหว่าง 0 ถึง 1 (หรือ 0% ถึง 100%)
  • ค่า Precision ยิ่งมากยิ่งดี (ค่าสูงสุดคือ 1 หรือ 100%)
  • Precision = 1 หมายถึงไม่มี False Positive เลย (ทุกครั้งที่ทำนายว่าเป็นคลาสบวก จะเป็นคลาสบวกจริง)
  • Precision = 0 หมายถึงไม่มี True Positive เลย (ทุกครั้งที่ทำนายว่าเป็นคลาสบวก จะเป็นคลาสลบจริง)

ข้อดี:

  • เหมาะสำหรับปัญหาที่ต้องการลดความผิดพลาดแบบ False Positive
  • ใช้วัดความน่าเชื่อถือของการทำนายคลาสบวก
  • ไม่ได้รับผลกระทบจากจำนวน True Negative

ข้อเสีย:

  • ไม่ได้พิจารณา False Negative
  • อาจทำให้เข้าใจผิดในกรณีที่มี Class Imbalance

การใช้งาน:

  • ใช้ในกรณีที่ต้นทุนของ False Positive สูง
  • เช่น การกรองสแปม (ต้องการให้แน่ใจว่าอีเมลที่ถูกทำนายว่าเป็นสแปมนั้นเป็นสแปมจริงๆ)
  • การวินิจฉัยโรค (ต้องการให้แน่ใจว่าผู้ป่วยที่ถูกวินิจฉัยว่าเป็นโรคนั้นเป็นโรคจริงๆ)

Recall (ความไว) - รายละเอียด

คำอธิบาย: Recall คือสัดส่วนของคลาสบวกจริงที่ถูกทำนายว่าเป็นคลาสบวก

สูตร:

Recall = TP / (TP + FN)

โดยที่:

  • TP (True Positive) คือจำนวนตัวอย่างที่ทำนายว่าเป็นคลาสบวก และเป็นคลาสบวกจริง
  • FN (False Negative) คือจำนวนตัวอย่างที่ทำนายว่าเป็นคลาสลบ แต่เป็นคลาสบวกจริง

การแปลผล:

  • ค่า Recall อยู่ระหว่าง 0 ถึง 1 (หรือ 0% ถึง 100%)
  • ค่า Recall ยิ่งมากยิ่งดี (ค่าสูงสุดคือ 1 หรือ 100%)
  • Recall = 1 หมายถึงไม่มี False Negative เลย (ทุกตัวอย่างที่เป็นคลาสบวกจริงถูกทำนายว่าเป็นคลาสบวก)
  • Recall = 0 หมายถึงไม่มี True Positive เลย (ทุกตัวอย่างที่เป็นคลาสบวกจริงถูกทำนายว่าเป็นคลาสลบ)

ข้อดี:

  • เหมาะสำหรับปัญหาที่ต้องการลดความผิดพลาดแบบ False Negative
  • ใช้วัดความสามารถในการค้นหาคลาสบวกทั้งหมด
  • ไม่ได้รับผลกระทบจากจำนวน True Negative

ข้อเสีย:

  • ไม่ได้พิจารณา False Positive
  • อาจทำให้เข้าใจผิดในกรณีที่มี Class Imbalance

การใช้งาน:

  • ใช้ในกรณีที่ต้นทุนของ False Negative สูง
  • เช่น การตรวจจับการฉ้อโกง (ต้องการตรวจจับการฉ้อโกงให้ได้มากที่สุด)
  • การวินิจฉัยโรคร้ายแรง (ต้องการตรวจพบผู้ป่วยที่เป็นโรคให้ได้มากที่สุด)

F1-score - รายละเอียด

คำอธิบาย: F1-score คือค่าเฉลี่ยฮาร์โมนิกของ Precision และ Recall

สูตร:

F1 = 2 * (Precision * Recall) / (Precision + Recall)

โดยที่:

  • Precision = TP / (TP + FP)
  • Recall = TP / (TP + FN)

การแปลผล:

  • ค่า F1-score อยู่ระหว่าง 0 ถึง 1 (หรือ 0% ถึง 100%)
  • ค่า F1-score ยิ่งมากยิ่งดี (ค่าสูงสุดคือ 1 หรือ 100%)
  • F1-score จะมีค่าสูงเมื่อทั้ง Precision และ Recall มีค่าสูง
  • F1-score จะมีค่าต่ำเมื่อ Precision หรือ Recall มีค่าต่ำ

ข้อดี:

  • พิจารณาทั้ง Precision และ Recall
  • เหมาะสำหรับปัญหาที่มีความไม่สมดุลของคลาส
  • ให้ความสำคัญกับค่าที่ต่ำระหว่าง Precision และ Recall

ข้อเสีย:

  • ไม่ได้พิจารณา True Negative
  • ให้น้ำหนักเท่ากันระหว่าง Precision และ Recall

การใช้งาน:

  • ใช้เมื่อต้องการสมดุลระหว่าง Precision และ Recall
  • ใช้ในปัญหาที่มีความไม่สมดุลของคลาส
  • ใช้เมื่อต้องการเมตริกเดียวที่สะท้อนทั้ง Precision และ Recall

ROC-AUC - รายละเอียด

คำอธิบาย: ROC-AUC คือพื้นที่ใต้กราฟ ROC (Receiver Operating Characteristic) ซึ่งแสดงความสัมพันธ์ระหว่าง True Positive Rate (TPR) และ False Positive Rate (FPR) ที่ threshold ต่างๆ

สูตร:

TPR = TP / (TP + FN) = Recall
FPR = FP / (FP + TN)
AUC = พื้นที่ใต้กราฟ ROC

การแปลผล:

  • ค่า AUC อยู่ระหว่าง 0 ถึง 1
  • AUC = 1 หมายถึงโมเดลสามารถแยกคลาสได้อย่างสมบูรณ์
  • AUC = 0.5 หมายถึงโมเดลทำนายไม่ดีไปกว่าการสุ่ม
  • AUC = 0 หมายถึงโมเดลทำนายผิดทั้งหมด (แต่สามารถกลับการทำนายเพื่อให้ได้ AUC = 1)

ข้อดี:

  • ไม่ขึ้นกับ threshold ที่เลือก
  • ทนทานต่อความไม่สมดุลของคลาส
  • วัดความสามารถในการจัดอันดับของโมเดล

ข้อเสีย:

  • ไม่ได้บอกถึงประสิทธิภาพที่ threshold ใดๆ โดยเฉพาะ
  • อาจทำให้เข้าใจผิดในกรณีที่มีความไม่สมดุลของคลาสมาก
